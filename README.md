# 🤖 Польза — Персональный ИИ-ассистент

Репозиторий содержит `workspace` (рабочую директорию, память, навыки и настройки) персонального ИИ-ассистента по имени Польза, работающего на базе платформы **OpenClaw**. Здесь же хранятся наработки по проекту «СПУТНИК», финансовые дашборды и скрипты автоматизации.

---

## 🚀 Как развернуть Пользу на новом сервере

Чтобы перенести меня на новый сервер и полностью восстановить мою память и навыки, выполните следующие шаги:

### Шаг 1. Подготовка сервера
Для работы OpenClaw потребуется **Node.js** (рекомендуется версия 22+).
Установите Node.js, если он еще не установлен:
```bash
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### Шаг 2. Установка платформы OpenClaw
Установите само ядро OpenClaw глобально через npm:
```bash
sudo npm install -g openclaw
```

### Шаг 3. Клонирование рабочей директории (Workspace)
OpenClaw по умолчанию ищет файлы ассистента в папке `~/.openclaw/workspace`. Нам нужно клонировать этот репозиторий именно туда:

```bash
# Создаем скрытую системную папку OpenClaw, если её нет
mkdir -p ~/.openclaw

# Клонируем репозиторий (потребуются SSH-ключи от GitHub)
git clone git@github.com:Steel-Eggs/Polsa.git ~/.openclaw/workspace
```
*(Если скачиваете по HTTPS: `git clone https://github.com/Steel-Eggs/Polsa.git ~/.openclaw/workspace`)*

### Шаг 4. Установка зависимостей скриптов
Зайдите в папку и установите пакеты, которые мы использовали для скриптов (например, для дашборда):
```bash
cd ~/.openclaw/workspace
npm install
```

### Шаг 5. Настройка ключей доступа (API)
Чтобы я смогла выйти в сеть и говорить, мне нужны ключи. Настройте конфигурацию платформы с помощью мастера:
```bash
openclaw configure
```
*(Вам понадобятся: токен Telegram-бота, ключ Google Gemini, и, при необходимости, ключ ElevenLabs, который прописывается в `~/.openclaw/.env`)*.

### Шаг 6. Запуск службы
Запустите системную службу (шлюз), чтобы я работала в фоне 24/7:
```bash
# Запуск через систему OpenClaw
openclaw gateway start

# Или если вы настраивали через systemd:
systemctl --user start openclaw-gateway.service
```

🎉 **Готово!** Как только шлюз запустится, напишите мне `/clear` в Telegram, чтобы я перечитала свои файлы из папки `memory/` и `MEMORY.md`, и я снова буду полностью в контексте всех наших дел!
