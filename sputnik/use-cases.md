# Сценарии для проекта «СПУТНИК» (B2C и B2B)

Формат записи для каждой ситуации:
- **Ситуация:** краткое описание реальной жизненной истории.
- **Боль/барьер:** почему человеку трудно решить это самому.
- **Как помогает ассистент:** конкретные действия ИИ (СПУТНИКа).
- **Медиума/формат:** куда можно упаковать кейс (шорт, статья, лендинг и т.д.).
- **Примечания:** дополнительные детали.

(список будет пополняться по мере получения сценариев)

## B2B / Профессиональные кейсы

- **Ситуация:** Работа маркетингового агентства с множеством узких задач (SEO, Контекст).
- **Боль/барьер:** Качественный анализ и сбор отчётов требует времени и узкой экспертизы. Один человек не успевает глубоко анализировать метрики по разным каналам.
- **Как помогает ассистент:** СПУТНИК работает как "Тимлид" — он создаёт и контролирует профильных субагентов (один только для SEO, другой только для Директа). Они собирают данные напрямую из кабинетов, а СПУТНИК формирует единый красивый отчёт для клиента.
- **Медиума/формат:** Лендинг для B2B, обучающее видео на YouTube.
- **Примечания:** Демонстрация мультиагентной архитектуры.

## B2C / Жизненные кейсы (Идеи для проработки)

*(Здесь мы будем собирать бытовые сценарии: медицина, юриспруденция, финансы, путешествия, питание. Например, анализ анализов крови, составление досудебной претензии, планирование отпуска с бронью билетов, составление меню на неделю из продуктов по скидке в ближайшем магазине и т.д. Раздел ждёт совместного мозгового штурма с Антоном)*

- **Ситуация (B2C: Финансы):** Пользователь хочет вести учёт личных и семейных финансов, но постоянно забывает вносить траты в таблицы или приложения.
- **Боль/барьер:** Ручной ввод каждой покупки из чека (категория, сумма, дата) отнимает время, вызывает лень и приводит к забрасыванию бюджета через неделю. Банковские приложения часто путают категории или не видят детализацию чека из супермаркета.
- **Как помогает ассистент (СПУТНИК):** Максимально бесшовный учёт. Человек просто делает фотографию чека прямо на кассе или кидает скриншот перевода из банка в чат. СПУТНИК сам распознаёт позиции с помощью Vision-модели, классифицирует товары (например, молоко -> продукты, бензин -> авто), суммирует скидки и заносит всё в личную базу данных. Пользователь в любой момент может открыть персональный интерактивный дашборд и увидеть графики трат по неделям/месяцам и долям категорий.
- **Медиума/формат:** Видео-демонстрация (Shorts/Reels) процесса "Сфоткал чек -> Сразу увидел обновленный график на дашборде".
- **Примечания:** Прямо сейчас этот функционал разрабатывается и тестируется нами в OpenClaw (связка CSV + Python + локальный Node.js сервер с дашбордом). Это готовый MVP для будущего модуля СПУТНИК Финансы.
