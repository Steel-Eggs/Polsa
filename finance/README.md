# Учет расходов Антона

## Структура
- `expenses_2026.csv` — помесячный список трат с разбивкой по товарным позициям и категориям.
- `expenses-data.json` / `expenses-data.js` — данные, подготовленные для дашборда (генерируются автоматически).
- `dashboard/` — мини-приложение с таблицей, фильтрами и графиками.
- `receipts/` — оригиналы чеков.
- При необходимости можно добавлять файлы `expenses_2027.csv` и т.д.

## Поля CSV
1. `date` — дата покупки (YYYY-MM-DD)
2. `vendor` — магазин/поставщик
3. `category` — верхний уровень (Продукты, Транспорт, и т.п.)
4. `subcategory` — уточнение внутри категории
5. `item` — позиция или описание траты
6. `quantity`, `unit_price_rub`, `line_total_rub` — детали по количеству и суммам
7. `vat_rate` — ставка НДС, если есть в чеке
8. `payment_method` — способ оплаты
9. `receipt_id` — идентификатор из чека (можно использовать для поиска оригинала)
10. `notes` — дополнительные пометки (скидки, кто пользовался, зачем покупали)

## Дашборд
- Файл запуска: `finance/dashboard/index.html`. Можно просто открыть его в браузере (данные подхватываются из `../expenses-data.js`) или поднять локальный сервер (`npx serve finance/dashboard`).
- Доступный функционал: фильтры по категориям/датам/поиску, таблица, суммы и графики по неделям (ISO) и месяцам.

## Обновление данных для интерфейса
1. Обнови/дополни CSV-файлы.
2. Запусти `python3 scripts/export_expenses_data.py` — скрипт соберёт все `expenses_*.csv`, пересоздаст `expenses-data.json` и `expenses-data.js`.
3. Перезагрузи страницу дашборда.

## Как пользоваться CSV
- Для новых чеков дописывай строки в `expenses_2026.csv`. При массовых тратях можно суммировать по чеку и указать позиции в `notes`.
- Если появляется новая категория — просто добавь её текстом; позже можно свести к единому справочнику.
- Сделанные фотографии чеков храним по умолчанию в `finance/receipts/` и в строке `notes` фиксируем путь.
